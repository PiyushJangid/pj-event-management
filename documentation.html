Introduction

PJ Event Management is a comprehensive WordPress plugin designed to create, manage, and display events on your website. Whether you need to showcase upcoming events, allow users to add events, or integrate events with Elementor, this plugin offers all the tools and flexibility you need.

Features

  • Event Management: Create, edit, and delete events with custom details like date, time, and location
  • Flexible Display Options: Show events in grids or lists with customizable layouts
  • Filtering Capabilities: Filter events by upcoming, past, or all events
  • Elementor Integration: Dedicated Elementor widget for seamless page builder integration
  • Shortcodes: Multiple shortcodes for displaying events anywhere on your site
  • Responsive Design: Mobile-friendly layouts that adapt to any screen size
  • User Authorization: Control who can manage events on your site
  • Customizable Templates: Easily style events to match your website's design

Installation

Follow these steps to install the PJ Event Management plugin:

  1. Download the plugin zip file
  2. Log in to your WordPress admin dashboard
  3. Navigate to Plugins → Add New
  4. Click the Upload Plugin button at the top of the page
  5. Select the downloaded zip file
  6. Click Install Now
  7. After installation completes, click Activate Plugin

Tip: After activation, you can access the plugin settings at Events → Settings in your WordPress admin menu.

Quick Start

Here's how to quickly start using the PJ Event Management plugin:

Display Events with Shortcode

Add this shortcode to any page or post to display upcoming events:

[pj_events]

Display Events with Elementor

  1. Edit a page with Elementor
  2. Search for "PJ Events" in the widget panel
  3. Drag the widget to your desired location
  4. Customize display options in the widget settings

Shortcodes

The plugin provides several shortcodes to display events in different ways:

Upcoming Events Shortcode

[pj_events]

This shortcode displays a grid of upcoming events with various customization options.

Parameters:

Parameter Description Default
per_page Number of events to display From plugin settings
columns Number of columns (1-4) 3
date_filter Filter events by date: "upcoming", "past", or "all" upcoming
pagination Pagination style: "standard", "load_more", "infinite", or "none" standard
show_date Display event date: "yes" or "no" yes
show_time Display event time: "yes" or "no" yes
show_location Display event location: "yes" or "no" yes
show_excerpt Display event excerpt: "yes" or "no" yes
show_filter_toggle Show filter buttons: "yes" or "no" no

Example:

[pj_events per_page="6" columns="3" show_filter_toggle="yes"]

Note: You can combine multiple parameters to create a highly customized event display.

Event Management Shortcode

[pj_all_events_management]

For admin users to manage events. This shortcode displays a table view of events with options to edit or delete them. It shows 10 events per page by default.

Parameters:

Parameter Description Default
per_page Number of events to display 10
pagination Pagination style standard

Example:

[pj_all_events_management per_page="15"]

Important: This shortcode will only work for users with the appropriate permissions to manage events.

Add/Edit Event Shortcode

[pj_add_edit_event]

Creates a form for adding or editing events. This is ideal for front-end event submission.

Elementor Widget

The plugin adds a "PJ Events" widget to Elementor that provides a visual way to add and configure events on your pages.

Content Options

  • Date Filter: Show upcoming, past, or all events
  • Posts Per Page: Number of events to display
  • Columns: Number of columns in the grid (1-4)
  • Pagination Type: Standard pagination, load more button, infinite scroll, or none
  • Show Filter Toggle: Enable filtering between upcoming, past, and all events
  • Display Options: Toggle visibility of date, time, location, and excerpt

Style Options

  • Title Color: Color for the widget title
  • Event Title Color: Color for individual event titles
  • Meta Info Color: Color for date, time, and location information
  • Button Colors: Customize button background and text colors
  • Spacing: Control spacing between events

Plugin Settings

Access plugin settings at Events → Settings in your WordPress admin.

General Settings

  • Events Per Page: Default number of events to display (used when not specified in shortcode)
  • Date Format: Format for displaying event dates
  • Time Format: Format for displaying event times
  • Enable Caching: Cache events to improve performance
  • User Roles: Select which user roles can manage events

Display Settings

  • Layout: Default layout for events display
  • Image Size: Default size for event featured images
  • Excerpt Length: Number of words to show in excerpts
  • Read More Text: Text for the "read more" button

Advanced Usage

For developers and advanced users who want to customize the plugin's functionality and appearance.

Custom Templates

To override the default templates, copy the template files from the plugin's templates directory to your theme in a folder named pj-event-management.

Example Structure:

your-theme/
├── pj-event-management/
│   ├── single-pj_event.php
│   ├── archive-pj_event.php
│   └── event-grid.php

JavaScript Events

The plugin triggers these events for developers:

  • pj_events_loaded - Fired after events are loaded
  • pj_events_filtered - Fired after events are filtered
  • pj_more_events_loaded - Fired after more events are loaded via AJAX

Example usage:

jQuery(document).on('pj_events_loaded', function() {
  console.log('Events have been loaded');
  // Your custom code here
});

Hooks and Filters

For developers, the plugin provides these hooks:

Actions:

  • pj_event_before_render - Before events are rendered
  • pj_event_after_render - After events are rendered
  • pj_event_before_form - Before the add/edit form is displayed
  • pj_event_after_form - After the add/edit form is displayed
  • pj_event_saved - After an event is saved

Filters:

  • pj_event_query_args - Modify event query arguments
  • pj_event_display_date - Modify the displayed date format
  • pj_event_display_time - Modify the displayed time format
  • pj_event_excerpt_length - Modify excerpt length
  • pj_event_meta_fields - Modify event meta fields

Example filter usage:

function my_custom_query_args($args) {
    // Modify the query arguments
    $args['posts_per_page'] = 12;
    return $args;
}
add_filter('pj_event_query_args', 'my_custom_query_args');

Troubleshooting

Common Issues

  1. Events not displaying

    Check if date filter is set correctly. If you're using "upcoming" filter, make sure you have events with future dates.

  2. Styling issues

    Make sure your theme CSS is not conflicting with the plugin styles. You can add custom CSS to override specific styles.

  3. Pagination not working

    Ensure permalinks are set correctly in WordPress settings. Try resetting permalinks by going to Settings → Permalinks and clicking Save.

  4. Images not displaying

    Check if featured images are set for your events. The plugin uses the featured image for the event thumbnail.

  5. Event deletion not working

    Clear browser cache or check the browser console for JavaScript errors. Ensure you have proper permissions to delete events.

  6. AJAX operations failing

    Verify nonce configuration and user permissions. Check browser console for any JavaScript errors that might be occurring.
